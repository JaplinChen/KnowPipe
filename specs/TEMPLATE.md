## 邊緣案例（Edge Cases）

各自生成獨立測試：

- [ ] EC-1：空輸入（`null` 或 `undefined`）應返回預設值，不拋出例外
- [ ] EC-2：超長輸入（>10,000 字元）應截短或返回錯誤訊息
- [ ] EC-3：CJK 字元（中文/日文/韓文）應正確處理
- [ ] EC-4：網路超時（> 10 秒）應返回 timeout 錯誤，不 hang
- [ ] EC-5：重複呼叫應為冪等（相同輸入，相同輸出）

---

## 不允許的行為（生成反向斷言）

- 不允許修改 `src/` 以外的系統狀態
- 不允許直接寫死路徑（應從 `.env` 讀取）
- 不允許靜默吞掉錯誤（所有錯誤必須拋出或 log）
- 不允許在 < N ms 內阻塞主執行緒
